Intro

Sky Knight is Shoot 'em up (shmup)  vertically scrolling top-down perspective shooter game. You need to defend the earth from the enemies attack, there are 7 weapons (bullet pattern) for help you each level, randomly dropped when the enemy destroyed. Can you survive from enemies attack? And fight the final boss to win this game.

Features:

  • 10 Challenging levels.
  • 7 Bullet patterns for player weapon attack.
  • 2 Bosses waiting and ready to Challenge you.
  • Using Construct 2 official plugins, without third party plugins,.
  • Game Pad supported (only for player shot & movement control).
  • Game assets included.
  • Cocoon.io free version supported.
  • Auto save level progess.
  • Mobile browser and mobile native optimized.
  • 1 Capx source code for desktop and mobile native version (Android & iOS) with auto detect config.

End Words

The game is fully compatible with all most common mobile devices. Sounds are enabled for mobile but we can't grant full audio compatibility on all mobile devices due to some well-know issue between some mobile-browser and HTML5.
 
If you face problems with this game, feel free to contact me by email ASAP. I will do my best to support you.

Project Structure

Layouts

1. Preloader layout.

2. Menu Screen layout.

3. 10 Level layouts.

4. Weapon test layout.

5. Objects Bank layout.

6. End Screen layout.

Event Sheets

1. Config Center: Variable for item drop chance, player start lives, boss 1 and 2 bonus score point and add your site URL.

2. End Screen: for end screen when player finisihed level 10.

3. Enemy AI: All about enemies AI you can find in this event sheet.

4. Global: default Data function and impact code.

5. HUD: Screen pause UI code, weapon test level UI code and Level UI code.

6. Items: Weapon power-up bullet patterns code.

7. Main: Main event sheet for level layout.

8. Menu: Menu screen code, detect device platform code, gamepad detection code, local storage check and load.

9. Music: music event sheet all about Level BGM you can find in this event sheet.

10. Player: Player movement control code and player weapon code.

11. Preloader: Event sheet for preload game resources.

12. Score: Save and load score code for highscore.

Instance Variables

Enemy Instance Variables

Each enemy have Instance Variables you can modified enemies hit points, trigger activation coordinate, etc. Click enemy you want to edit then look at Properties bar on Instance variables column.

 If you change Points then you must edit "On Enemy_Destroy" function on "Enemy_AI" event sheet line 9.

Export Game

Before export process make sure review these settings below.

HTML5 Platform

Click "Export Project" then select "HTML5 Platform" then click next button and make sure "minify script" checked then select template style: Normal style, Advert bar style or Embed style then click export. After export process finished you can upload the result to you site or HTML5 platform site.

Mobile Native Platform

Check "Use new Intel XDK project format" if you want export the game using Intel XDK.

Uncheck "Use new Intel XDK project format" if you want export the game using Cocoon.io. I recommened you to use Cocoon.io for better performance.

Important: 
 
Please add these line under <access origin="*"/> in config.xml
<allow-navigation href="*"/>


<allow-intent href="*"/>
 
This is for enable external link access on Android platform. See example picture below.
 
 
If you need guide for compile using cocoon.io please visit this link:
https://docs.cocoon.io/article/how-to-export-android-in-construct-2-with-cocoon-io/
or
https://shatter-box.com/knowledgebase/cocoonio/

For use Intel XDK for export to mobile platform, please check this tutotial link:
https://shatter-box.com/knowledgebase/intel-xdk-complete-guide-for-construct-2/
